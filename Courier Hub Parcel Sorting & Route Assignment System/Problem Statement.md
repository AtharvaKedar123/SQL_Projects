36. Courier Hub Parcel Sorting & Route Assignment System
Detailed Problem Statement

A courier company wants to manage parcels, courier hubs, sorting belts, delivery routes, courier agents, and parcel delivery status using SQL.

Parcels arrive at different courier hubs, move through sorting belts, get assigned to delivery routes, and are finally handled by courier agents. The company wants to track parcel movement, identify delayed parcels, monitor sorting belt usage, manage route assignments, and analyze courier agent performance.

The system should store:

Parcel details
Sender details
Receiver details
Courier hub details
Sorting belt details
Delivery route details
Courier agent details
Parcel sorting records
Route assignment details
Parcel delivery status

The system should help the courier company answer questions like:

Which courier hub handled the most parcels?
Which sorting belt processed the most parcels?
Which courier agent delivered the most parcels?
Which parcels are delayed?
Which routes have the highest parcel load?
How many parcels are pending, in transit, and delivered?
Which parcels are high priority?
What is the total delivery fee collected?
