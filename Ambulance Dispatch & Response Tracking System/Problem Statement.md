Problem 23: Smart Ambulance Dispatch & Response Tracking System
Detailed Problem Statement

A hospital network wants to manage ambulance dispatch, emergency calls, ambulance availability, drivers, patients, and response time digitally using SQL.

When an emergency call is received, the nearest available ambulance is assigned to the patient. The system should track ambulance location zone, driver details, patient condition, dispatch time, arrival time, hospital destination, and case status.

The system should store:

Patient details
Emergency call details
Ambulance details
Driver details
Hospital details
Dispatch records
Response time and case status

The system should help the hospital answer questions like:

Which ambulances are currently available?
Which emergency calls are pending?
Which ambulance handled the most cases?
Which cases had delayed response?
Which hospital received the most emergency patients?
What is the average ambulance response time?
