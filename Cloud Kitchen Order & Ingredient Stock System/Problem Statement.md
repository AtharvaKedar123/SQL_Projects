32. Cloud Kitchen Order & Ingredient Stock System
Detailed Problem Statement

A cloud kitchen wants to manage its food orders, menu items, recipes, ingredients, stock levels, preparation status, and delivery orders using SQL.

Customers place food orders through online platforms. Each menu item requires different ingredients, and the kitchen must track whether enough stock is available to prepare orders. The management wants to monitor popular dishes, low-stock ingredients, order status, delivery performance, and total revenue.

The system should store:

Customer details
Menu item details
Ingredient details
Recipe ingredient requirements
Order details
Order item details
Ingredient stock levels
Delivery details
Preparation status

The system should help the cloud kitchen answer questions like:

Which menu items are ordered the most?
Which ingredients are running low?
How much revenue was generated from completed orders?
Which orders are still being prepared?
Which delivery partners handled the most deliveries?
Which customers placed the most orders?
Which menu items require the highest number of ingredients?
Which orders are delayed?
