Problem 43: Airport Baggage Tracking & Lost Luggage System
Detailed Problem Statement

An airport wants to manage its baggage tracking and lost luggage system digitally using SQL.

The airport has multiple passengers, flights, baggage tags, scan checkpoints, baggage movement records, lost luggage complaints, and recovery details. Each passenger can carry one or more baggage items. Every baggage item is assigned a unique baggage tag. The baggage is scanned at different checkpoints such as check-in counter, security scan, loading area, aircraft, arrival belt, and baggage claim.

The system should also track whether baggage is loaded, delayed, missing, recovered, or delivered to the passenger. If a passenger reports lost luggage, the system should store complaint details and recovery status.

The system should store:

Passenger details
Flight details
Baggage tag details
Airport checkpoint details
Baggage scan records
Lost luggage complaint details
Recovery status
Delivery status

The system should help the airport answer questions like:

Which baggage items are currently missing?
Which passenger reported lost luggage?
Which flight has the most baggage complaints?
At which checkpoint was baggage last scanned?
How many bags were successfully delivered?
Which lost bags were recovered?
Which baggage items are still pending recovery?
