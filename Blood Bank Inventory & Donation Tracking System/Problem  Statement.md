Detailed Problem Statement

A blood bank wants to manage blood donations, blood inventory, hospital requests, and blood unit expiry digitally using SQL.

Donors donate blood of different blood groups. Every donated blood unit has a collection date, expiry date, storage location, and availability status. Hospitals can request blood units based on blood group and quantity. The blood bank must track which requests are fulfilled, which blood units are issued, and which units are close to expiry.

The system should store:

Donor details
Blood group details
Donation records
Blood unit inventory
Hospital details
Hospital blood requests
Blood issue details

The system should help the blood bank answer questions like:

Which blood units are available?
Which blood units are expired?
Which blood group has the lowest stock?
Which donors donated the most?
Which hospital requests are pending?
How many units were issued to each hospital?
