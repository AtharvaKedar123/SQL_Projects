Problem 6: E-Commerce Return & Refund Tracking System
Detailed Problem Statement

An e-commerce company wants to manage product returns and refunds digitally using SQL.

Customers place orders for different products. Sometimes customers return products because of reasons like damaged product, wrong size, defective item, late delivery, or wrong product delivered. After a return request is raised, the company checks the return status and processes refunds based on the return approval.

The system should store:

Customer details
Product details
Order details
Ordered item details
Return request details
Refund details
Return reason and refund status

The system should help the company answer questions like:

Which products are returned the most?
Which customers requested the most returns?
Which refunds are still pending?
What is the total refunded amount?
What are the most common return reasons?
Which product category has the highest return count?
