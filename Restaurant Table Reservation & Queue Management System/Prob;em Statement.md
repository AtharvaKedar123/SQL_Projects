Problem 16: Restaurant Table Reservation & Queue Management System
Detailed Problem Statement

A restaurant wants to manage table reservations, walk-in customers, waiting queue, dining sessions, and billing digitally using SQL.

Customers can reserve tables in advance or join a queue as walk-in customers. The restaurant has different table types such as two-seater, four-seater, family table, and VIP table. The system should track table availability, reservation status, queue position, dining start/end time, and payment details.

The system should store:

Customer details
Restaurant table details
Reservation details
Walk-in queue details
Dining session details
Billing and payment details

The system should help the restaurant answer questions like:

Which tables are currently available?
Which reservations are confirmed today?
Which customers are waiting in queue?
Which tables are currently occupied?
How much revenue was generated?
Which table type is used the most?
