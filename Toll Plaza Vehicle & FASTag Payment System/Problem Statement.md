33. Smart Toll Plaza Vehicle & FASTag Payment System
Detailed Problem Statement

A smart toll plaza wants to manage vehicles, toll booths, FASTag accounts, toll payments, vehicle categories, lane usage, and unpaid toll records using SQL.

Vehicles pass through different toll lanes every day. Some vehicles pay through FASTag, some have insufficient balance, and some payments remain unpaid. The toll authority wants to track vehicle movement, calculate toll revenue, identify unpaid tolls, monitor booth usage, and analyze vehicle category traffic.

The system should store:

Vehicle details
Vehicle category details
FASTag account details
Toll booth details
Lane usage details
Toll transaction details
Payment status
Unpaid toll records

The system should help the toll authority answer questions like:

Which vehicle category uses the toll plaza the most?
Which toll booth collected the highest revenue?
Which vehicles have unpaid tolls?
Which FASTag accounts have low balance?
How much total toll revenue was collected?
Which toll lane is used the most?
Which vehicles passed through the toll plaza most often?
Which transactions failed due to insufficient FASTag balance?
