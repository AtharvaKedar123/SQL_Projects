Detailed Problem Statement

A college placement department wants to manage campus placements digitally using SQL.

The college has multiple students from different branches. Different companies visit the campus and offer different job roles with salary packages. Students apply for these roles based on eligibility criteria such as CGPA, branch, and graduation year. After applying, students go through multiple interview rounds like aptitude test, technical round, coding round, managerial round, and HR round.

The system should store:

Student details
Company details
Job role details
Student applications
Interview round details
Final selection status
Salary package details

The system should help the placement department answer questions like:

Which students got selected?
Which companies hired the most students?
Which students are still in the interview process?
What is the highest package offered?
How many students applied for each company?
Which branch has the highest number of selected students?
