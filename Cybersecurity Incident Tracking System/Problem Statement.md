Problem 9: Cybersecurity Incident Tracking System
Detailed Problem Statement

An organization wants to manage cybersecurity incidents digitally using SQL.

The company has multiple employees using different devices such as laptops, desktops, and company phones. Sometimes these devices face security threats like phishing attacks, malware infection, ransomware attempts, suspicious login attempts, or data leakage. The security team needs to record every incident, track severity, assign resolution status, and monitor which departments or devices are most vulnerable.

The system should store:

Employee details
Department details
Device details
Threat type details
Security incident details
Resolution details
Incident status

The system should help the security team answer questions like:

Which incidents are still open?
Which department has the highest number of incidents?
Which threat type occurs the most?
Which devices are affected repeatedly?
How many critical incidents happened?
Which incidents have been resolved?
