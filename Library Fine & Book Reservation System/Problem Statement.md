Problem 8: Library Fine & Book Reservation System
Detailed Problem Statement

A library wants to manage books, members, book issues, returns, reservations, and fines digitally using SQL.

Members can borrow books from the library for a limited number of days. If a book is returned late, a fine is charged. Members can also reserve books that are currently unavailable. The library needs to track available copies, issued books, overdue books, pending reservations, and fine payments.

The system should store:

Library member details
Book details
Book issue details
Book return details
Book reservation details
Fine details
Payment status

The system should help the librarian answer questions like:

Which books are currently issued?
Which books are overdue?
Which members have unpaid fines?
Which books are reserved the most?
Which books are available right now?
How much fine amount has been collected?
